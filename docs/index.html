Get Started View on GitHub 
-- Traditional SQL
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255),
    embedding VECTOR(768)
);

-- Native Tensors
CREATE TENSOR embeddings (
    shape [1000, 768],
    dtype float32
);

-- Mixed Queries
SELECT u.name, 
       cosine_similarity(e.embeddings, query_vector) as similarity
FROM users u 
JOIN embeddings e ON u.id = e.user_id
WHERE similarity > 0.8;

Why TelumDB?

The Problem with Current Solutions

Modern applications need both traditional data storage and AI/ML capabilities. Today's solutions force you to choose between general-purpose databases (like PostgreSQL) that lack native tensor support, or specialized vector databases that can't handle traditional workloads.

The TelumDB Solution

TelumDB eliminates this compromise by providing a single system that excels at both. Store your user data, transactions, and embeddings in one place. Query them together with familiar SQL syntax enhanced with powerful tensor operations.

Real-World Benefits

  • Simplified Architecture: One database instead of multiple specialized systems
  • Reduced Complexity: No data synchronization between different storage systems
  • Lower Costs: Single system to deploy, maintain, and scale
  • Better Performance: Native mixed queries without cross-system joins
  • Familiar Development: Use existing SQL knowledge with minimal learning curve

Get Started

Option 1: Build from Source

git clone https://github.com/telumdb/telumdb.git
cd telumdb
make build
./telumdb

Option 2: Docker

docker run -d \
  --name telumdb \
  -p 5432:5432 \
  -p 8080:8080 \
  -v telumdb_data:/app/data \
  telumdb/telumdb:latest

Quick Test

Start the server, then use the CLI in another terminal:

# Start server
./telumdb

# In another terminal
./telumdb-cli

# Try these commands
telumdb> CREATE TABLE users (id INTEGER, name VARCHAR(255));
telumdb> CREATE TENSOR embeddings (shape [1000, 768], dtype float32);
telumdb> SHOW TABLES;
telumdb> SHOW TENSORS;
