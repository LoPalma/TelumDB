Getting Started with TelumDB

Installation

Option 1: Build from Source

Build TelumDB from source for the latest features:

# Clone repository
git clone https://github.com/telumdb/telumdb.git
cd telumdb

# Build binaries
make build

# Verify installation
./telumdb -version
./telumdb-cli -version

Option 2: Docker

Run TelumDB in a Docker container:

# Pull latest image
docker pull telumdb/telumdb:latest

# Run container
docker run -d \
  --name telumdb \
  -p 5432:5432 \
  -p 8080:8080 \
  -v telumdb_data:/app/data \
  telumdb/telumdb:latest

Option 3: Binary Download

Download pre-compiled binaries for your platform:

# Linux (AMD64)
wget https://github.com/telumdb/telumdb/releases/latest/download/telumdb-linux-amd64
chmod +x telumdb-linux-amd64

# macOS (AMD64)
wget https://github.com/telumdb/telumdb/releases/latest/download/telumdb-darwin-amd64
chmod +x telumdb-darwin-amd64

# Windows (AMD64)
# Download telumdb-windows-amd64.exe from releases page

Configuration

TelumDB uses a YAML configuration file. Create a config.yaml file:

server:
  host: "0.0.0.0"
  port: 5432
  http_port: 8080
  max_connections: 1000

storage:
  data_dir: "./data"
  engine: "hybrid"
  cache_size: 1073741824  # 1GB
  tensor:
    chunk_size: [64, 64, 64]
    default_dtype: "float32"
    memory_limit: 4294967296  # 4GB

logging:
  level: "info"
  format: "json"
  output: "stdout"

metrics:
  enabled: true
  port: 9000

Environment Variables

You can also configure TelumDB using environment variables:

export TELUMDB_CONFIG_FILE=/path/to/config.yaml
export TELUMDB_DATA_DIR=/path/to/data
export TELUMDB_LOG_LEVEL=debug
export TELUMDB_PORT=5432

Basic Usage

Starting the Server

Start the TelumDB server:

# Use default configuration
./telumdb

# Use custom configuration
./telumdb -config /path/to/config.yaml

# Check version
./telumdb -version

The server will start on:

  • Database protocol: localhost:5432
  • HTTP API: localhost:8080
  • Metrics: localhost:9000

Using the CLI

Connect to TelumDB using the CLI tool:

# Interactive mode
./telumdb-cli

# Execute single command
./telumdb-cli -c "SHOW TABLES"

# Execute script file
./telumdb-cli -f script.tql

# Connect to remote server
./telumdb-cli -url telumdb://remote-host:5432

SQL Operations

TelumDB supports standard SQL operations for traditional data:

Create Tables

CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE,
    age INTEGER,
    department VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Insert Data

INSERT INTO users (name, email, age, department) VALUES
    ('Alice Johnson', 'alice@company.com', 30, 'Engineering'),
    ('Bob Smith', 'bob@company.com', 25, 'Marketing'),
    ('Charlie Brown', 'charlie@company.com', 35, 'Engineering');

Query Data

-- Basic SELECT
SELECT * FROM users WHERE age > 25;

-- Complex query with JOINs
SELECT u.name, u.department, COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.created_at >= '2023-01-01'
GROUP BY u.id, u.name, u.department
ORDER BY order_count DESC;

Tensor Operations

TelumDB provides native tensor operations for AI/ML workloads:

Create Tensors

CREATE TENSOR user_embeddings (
    shape [10000, 768],
    dtype float32,
    chunk_size [1000, 768]
);

CREATE TENSOR model_weights (
    shape [768, 256],
    dtype float32,
    chunk_size [768, 256]
);

Tensor Operations

-- Mathematical operations
ADD(embeddings, bias_vector);
MULTIPLY(embeddings, scale_factor);
MATRIX_MULTIPLY(embeddings, weights);

-- Linear algebra
TRANSPOSE(embeddings);
SVD(covariance_matrix);
EIGENVALUES(correlation_matrix);

-- Reductions
SUM(embeddings, axis=0);
MEAN(embeddings, axis=1);
MAX(embeddings);
MIN(embeddings);

-- Activation functions
RELU(hidden_layer);
SIGMOID(output_layer);
TANH(activation_layer);

-- Convolution
CONV1D(input_signal, kernel_1d, stride=2, padding=1);
CONV2D(input_image, kernel_2d, stride=[2,2], padding=[1,1]);

-- Similarity
COSINE_SIMILARITY(vector_a, vector_b);

Mixed Queries

The power of TelumDB comes from combining SQL and tensor operations:

Similarity Search

-- Find users with similar embeddings
SELECT u.name, u.email, u.department,
       cosine_similarity(ue.embeddings, query_vector) as similarity
FROM users u
JOIN user_embeddings ue ON u.id = ue.user_id
WHERE u.department = 'Engineering'
  AND cosine_similarity(ue.embeddings, query_vector) > 0.8
ORDER BY similarity DESC
LIMIT 10;

Product Recommendations

-- Recommend products based on user embeddings
SELECT p.name, p.category, p.price,
       cosine_similarity(pe.embeddings, user_profile) as similarity
FROM products p
JOIN product_embeddings pe ON p.id = pe.product_id
WHERE p.category = 'Electronics'
  AND cosine_similarity(pe.embeddings, user_profile) > 0.7
ORDER BY similarity DESC, p.price ASC
LIMIT 5;

Complex Analytics

-- User segmentation with tensor clustering
SELECT u.department,
       AVG(cosine_similarity(ue.embeddings, dept_centroid)) as avg_similarity,
       COUNT(*) as user_count
FROM users u
JOIN user_embeddings ue ON u.id = ue.user_id
GROUP BY u.department
HAVING COUNT(*) > 5
ORDER BY avg_similarity DESC;

Troubleshooting

Connection Issues

Problem: "Connection refused" error

Solution: Make sure the TelumDB server is running:

# Check if server is running
./telumdb

# In another terminal, test connection
./telumdb-cli -c "SELECT 1"

Port Conflicts

Problem: "Address already in use" error

Solution: Change the port in configuration:

server:
  port: 5433  # Use different port

Permission Issues

Problem: "Permission denied" error

Solution: Check data directory permissions:

# Create data directory with proper permissions
mkdir -p ./data
chmod 755 ./data

# Or specify different data directory
./telumdb -config config.yaml
