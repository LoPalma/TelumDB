SQL Syntax

TelumDB supports standard SQL commands for traditional relational data operations.

Data Definition Language (DDL)

CREATE TABLE

CREATE TABLE table_name ( column_name data_type [NULL | NOT NULL] [DEFAULT default_value], ... );
CREATE TABLE users ( id INTEGER NOT NULL, name VARCHAR(255) NOT NULL, email VARCHAR(255), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

DROP TABLE

DROP TABLE table_name;

Data Manipulation Language (DML)

INSERT

INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);

SELECT

SELECT column1, column2, ... FROM table_name [WHERE condition] [ORDER BY column] [LIMIT n];

UPDATE

UPDATE table_name SET column1 = value1, column2 = value2, ... [WHERE condition];

DELETE

DELETE FROM table_name [WHERE condition];

Utility Commands

SHOW TABLES

SHOW TABLES;

DESCRIBE

DESCRIBE table_name;

TQL (Tensor Query Language)

TelumDB Query Language extends SQL with specialized tensor operations for machine learning and scientific computing.

Tensor Definition

CREATE TENSOR

CREATE TENSOR tensor_name ( shape [dim1, dim2, ...], dtype data_type [, chunk_size [chunk1, chunk2, ...]] );
CREATE TENSOR embeddings ( shape [1000, 768], dtype float32, chunk_size [100, 768] );
Note: Chunk size is optional but recommended for large tensors to optimize memory usage and I/O operations.

DROP TENSOR

DROP TENSOR tensor_name;

Tensor Information

SHOW TENSORS

SHOW TENSORS;

DESCRIBE TENSOR

DESCRIBE TENSOR tensor_name;

Tensor Operations

Tensor operations can be used as standalone statements or within SELECT queries.

OPERATION(tensor_name [, parameters...]);
-- Standalone operations SUM(embeddings, axis=1); MEAN(embeddings); MAX(embeddings, axis=0); -- In queries SELECT COSINE_SIMILARITY(vec1, vec2) FROM vectors;

Tensor Operations

Mathematical Operations

ADD(tensor1, tensor2)

Element-wise addition with broadcasting support

MULTIPLY(tensor1, tensor2)

Element-wise multiplication with broadcasting support

MATRIX_MULTIPLY(tensor1, tensor2)

Matrix multiplication for 2D tensors

TRANSPOSE(tensor)

Transpose operation for 2D tensors

Reduction Operations

SUM(tensor[, axis=n])

Sum of elements along specified axis (default: all)

MEAN(tensor[, axis=n])

Mean of elements along specified axis (default: all)

MAX(tensor[, axis=n])

Maximum value along specified axis (default: all)

MIN(tensor[, axis=n])

Minimum value along specified axis (default: all)

Activation Functions

RELU(tensor)

Rectified Linear Unit: max(0, x)

SIGMOID(tensor)

Sigmoid activation: 1/(1 + e^(-x))

TANH(tensor)

Hyperbolic tangent activation

Convolution Operations

CONV1D(input, kernel[, stride=n][, padding=n])

1D convolution with optional stride and padding

CONV2D(input, kernel[, stride=[h,w]][, padding=[h,w]])

2D convolution with optional stride and padding

Linear Algebra

SVD(tensor)

Singular Value Decomposition

EIGENVALUES(tensor)

Eigenvalue computation for square matrices

COSINE_SIMILARITY(tensor1, tensor2)

Cosine similarity between two tensors

EUCLIDEAN_DISTANCE(tensor1, tensor2)

Euclidean distance between two tensors

Tensor Manipulation

TENSOR_SLICE(tensor, ranges...)

Extract slice along specified dimensions

TENSOR_RESHAPE(tensor, new_shape)

Reshape tensor to new dimensions

Operation Examples

-- Create tensors
CREATE TENSOR A (shape [3, 4], dtype float32);
CREATE TENSOR B (shape [4, 2], dtype float32);

-- Matrix multiplication
MATRIX_MULTIPLY(A, B);

-- Element-wise operations
ADD(A, B);
MULTIPLY(A, 2.5);

-- Reductions
SUM(A, axis=1);  -- Sum along columns
MEAN(A);         -- Global mean

-- Activation functions
RELU(A);
SIGMOID(A);

-- Convolution
CONV2D(input_tensor, kernel_tensor, stride=[2,2], padding=[1,1]);

-- Similarity
COSINE_SIMILARITY(vec1, vec2);
EUCLIDEAN_DISTANCE(point1, point2);

Meta Commands

Meta commands are CLI-specific commands that start with backslash (\) and provide database management utilities.

Command Alias Description Example
\help \h Show help message \help
\quit \q Exit the CLI \quit
\list \l List all databases \list
\tables \dt List all tables \tables
\tensors \dT List all tensors \tensors
\describe \d Describe table or tensor \describe users
\connect \c Connect to database \connect mydb

Connection URLs

TelumDB uses URL-style connection strings for database connections.

URL Format

telumdb://[user[:password]@]host[:port][/database][?parameter=value&...]

Components

Component Required Description Example
scheme Yes Always "telumdb" telumdb
user No Username for authentication admin
password No Password for authentication secret123
host Yes Server hostname or IP localhost
port No Server port (default: 5432) 5432
database No Database name mydb

Examples

Basic Connection

telumdb://localhost:5432

With Authentication

telumdb://user:password@localhost:5432

With Database

telumdb://user:password@localhost:5432/mydb

Remote Server

telumdb://admin@db.example.com:5432/production

Error Codes

Common error codes and their meanings in TelumDB.

Code Message Cause Solution
Connection Error "failed to connect to server" Server not running or wrong host/port Check server status and connection URL
Parse Error "Statement must end with semicolon" Missing semicolon in SQL/TQL Add semicolon at end of statement
Validation Error "Invalid tensor shape format" Malformed CREATE TENSOR syntax Use comma-separated integers for shape
Runtime Error "tensor dimensions incompatible" Operation shape mismatch Check tensor shapes before operations
Memory Error "slice too large" Requested slice exceeds limits Use smaller slice or increase limits
